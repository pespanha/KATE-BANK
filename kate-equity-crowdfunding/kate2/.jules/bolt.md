## 2024-05-15 - DB Aggregation vs JS Reduce\n**Learning:** Prisma `.reduce()` on included relations is an anti-pattern here because it pulls full datasets into Node.js memory. Prisma's `aggregate()` offloads the computation to the database, saving memory and network latency.\n**Action:** Always prefer `prisma.reservation.aggregate({ _sum: { ... } })` over fetching arrays of records just to calculate sums, especially as platform usage grows.
